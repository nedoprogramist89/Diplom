1. Тема ДП(Р): «Разработка информационной системы для прохождения соревнований и хакатонов «CompetitionHub» (на примере ФГБОУ ВО «Российский экономический университет имени Г.В. Плеханова»)».
Утверждена (распоряжением) от «06» апреля 2026 года № 18.01-17-156.

2. Дата выдачи задания: «06» апреля 2026 г.

3. Исходные данные к работе (цель, задачи и объём исследования, предполагаемые методы и методики исследования и т.д.)

3.1. Цель

Автоматизировать организацию и проведение соревнований по программированию и хакатонов (регистрация участников, работа с задачами и решениями, итоги и отчётность). Обеспечить надёжное хранение данных и разграничение прав пользователей. Доступ к данным и функциям в системе обеспечивается через ролевую политику (участник, организатор, жюри, персонал панели управления).

**Задачи дипломного проекта:**

1) выполнить анализ предметной области проведения олимпиад и хакатонов в образовательной организации;
2) разработать внешнюю спецификацию и спроектировать архитектуру приложения, логическую и физическую модели данных, структурную, функциональную схемы, схему пользовательского интерфейса, DFD;
3) реализовать серверную часть на Django, веб-интерфейс (DTL, HTML, CSS, JavaScript), модули accounts, competitions, hackathons;
4) провести тестовые испытания и оформить программную документацию (текст программы, сценарий испытаний, руководство пользователя, скрипт базы данных).

**Методы и методики:** объектно-ориентированное проектирование и программирование; моделирование UML (диаграмма прецедентов, диаграмма классов); проектирование реляционной базы данных (ER-модель, нормализация); разработка по многослойной архитектуре «клиент — сервер»; тестирование по сценариям (функциональное, модульное, интеграционное, GUI); оценка качества по ISO/IEC 25010.

**Объём исследования:** разработка полнофункционального прототипа информационной системы для ФГБОУ ВО «РЭУ им. Г.В. Плеханова» (Московский приборостроительный техникум) с пояснительной запиской объёмом 35–40 страниц без приложений и комплектом приложений по методическим материалам.


3.2. Технологии

Технологии: веб-платформа: Python (3.11), Django (5.2) с поддержкой MVT, Django Template Language, HTML5, CSS3, JavaScript, Tailwind CSS (3); база данных: PostgreSQL (14), SQLite (3) — для отладки; API: Django REST Framework (3.14), Simple JWT (5.3); сервер приложений: Gunicorn (21).


3.3. Инструменты

Инструменты: Visual Studio Code (1.95), Git (2.42), Python pip и venv (3.11), pgAdmin 4 (8) / DBeaver (24), PostgreSQL (14), Microsoft Word 2024, draw.io, Google Chrome (131), встроенный сервер Django — manage.py runserver, «Ножницы» (Snipping Tool) Windows 11.


3.4. Входные данные

Входные данные: данные пользователя (логин, пароль, e-mail, имя, фамилия, организация, роль, тема оформления), данные соревнования (название, описание, статус, даты начала и окончания, аудитория, уровень, квота участников), данные задачи (предмет, тип задания, название, условие, баллы, эталонный ответ, варианты ответа или пары соответствия, файл материала), данные решения (текст или выбранные варианты, файл вложения, статус, баллы, комментарий проверяющего), данные участия (пользователь, соревнование, дата регистрации), данные объявления (заголовок, текст), данные хакатона (название, описание, этап, регистрация, команда, заявка в команду, сообщение чата), параметры поиска и фильтрации в списках, учётные данные и JSON-запросы REST API, настройки подключения к БД и почты в файле окружения (.env).


3.5. Требования

WEB:
Реализовать регистрацию и авторизацию по ролям: участник, организатор, жюри;
Реализовать смену темы оформления (светлая и тёмная);
Реализовать восстановление пароля по ссылке на электронную почту;
Реализовать просмотр главной страницы, публичных итогов и архива мероприятий;
Реализовать просмотр списка соревнований с поиском и фильтрацией;
Реализовать просмотр карточки соревнования, списка задач и карточки задачи;
Реализовать регистрацию на соревнование и отмену регистрации в период «Регистрация»;
Реализовать отправку решения в период «Идёт» (краткий ответ, выбор вариантов, соответствие, развёрнутый ответ, файл);
Реализовать автоматическую и ручную проверку решений;
Реализовать просмотр своих решений, таблицы результатов соревнования и экспорт результатов в CSV;
Реализовать просмотр списка хакатонов, регистрацию на хакатон, работу с командами и чатом;
Реализовать профиль пользователя, список уведомлений и адаптивную навигацию (боковая панель).

Участник:
Реализовать регистрацию в системе, вход и выход из учётной записи;
Реализовать просмотр каталога соревнований и хакатонов;
Реализовать регистрацию на соревнование и хакатон, отмену регистрации в допустимый этап;
Реализовать отправку решений по задачам и просмотр своих решений;
Реализовать просмотр таблицы результатов соревнования;
Реализовать участие в команде хакатона (создание команды, заявка в команду, чат);
Реализовать просмотр и редактирование профиля, смену пароля и темы оформления.

Организатор:
Реализовать создание, изменение и удаление своих соревнований и хакатонов;
Реализовать смену этапа соревнования и хакатона;
Реализовать добавление, изменение и удаление задач, вариантов ответа и объявлений по своим соревнованиям;
Реализовать просмотр списка участников своего соревнования;
Реализовать просмотр всех решений по задаче, выставление баллов, статуса и комментария;
Реализовать экспорт в CSV (результаты, участники, решения — по предусмотренным формам);
Реализовать управление командами и участниками хакатона, просмотр чатов.

Жюри:
Реализовать просмотр участников и решений по любым соревнованиям;
Реализовать оценку решений (баллы, статус, комментарий);
Реализовать просмотр результатов и экспорт таблицы результатов в CSV;
Реализовать ограничение: без создания и редактирования чужих соревнований и задач.

Администратор (персонал панели управления):
Реализовать вход в панель управления по адресу /admin/;
Реализовать добавление, изменение и удаление пользователей (назначение роли, активности);
Реализовать управление соревнованиями, задачами, вариантами ответа, участиями, решениями, объявлениями и справочниками;
Реализовать просмотр журнала аудита действий в панели управления.

REST API:
Реализовать получение JWT-токена по учётным данным пользователя;
Реализовать программный доступ к данным соревнований, задач, решений и учётных записей через REST API.
